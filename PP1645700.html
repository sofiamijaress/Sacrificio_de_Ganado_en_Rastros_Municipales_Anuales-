Otro de los problemas ms bsicos es la colinealidad, esta ocurre cuando dos o ms variables independientes estn fuertemente correlacionadas entre s, lo que significa que aportan informacin muy similar. Esto puede provocar que los coeficientes de la regresin se vuelvan inestables, difciles de interpretar e incluso influyan negativamente en la capacidad predictiva del modelo.

Por eso, utilic un heatmap para visualizar las correlaciones y apliqu un umbral de 0.85 para identificar los pares de variables con correlacin muy alta. Despus, program un ciclo que compara cada par de variables colineales y decide cul conservar en funcin de su relacin con mi variable de salida PRECIO_MEDIO. De esta manera, elimino la variable menos relevante y reduzco el riesgo de colinealidad.

sns.heatmap(cor, cmap="coolwarm", annot=cor.applymap(lambda x: f"{x:.2f}" if x >= 0.85 else " "), fmt="", cbar=True)
# Obtener los ndices (i,j) de la posicin del valor mximo
i, j = np.unravel_index(cor.values.argmax(), cor.shape)
y = df.PRECIO_MEDIO
corr_matrix = df.corr().abs()
to_drop = set()
for i in range(len(corr_matrix.columns)):
    for j in range(i):
        if corr_matrix.iloc[i, j] > 0.85:
            colname1 = corr_matrix.columns[i]
            colname2 = corr_matrix.columns[j]
            if abs(df[colname1].corr(y)) > abs(df[colname2].corr(y)):
                to_drop.add(colname2)
            else:
                to_drop.add(colname1)
print("Columnas a eliminar por prueba de colinealidad:", to_drop)
df = df.drop(to_drop,axis = 1)

Continuamos con la metodologa de seleccin de caractersticas, y es que no todas las variables disponibles aportan informacin til ya que algunas pueden ser irrelevantes y otras estar tan correlacionadas entre s que generan redundancia y problemas de colinealidad. Entonces al reducir el nmero de predictores a un subconjunto ms representativo, busco que el modelo sea ms estable e interpretable.

Esta metodologa evala distintas combinaciones de variables de manera iterativa y se queda con aquellas que maximizan una mtrica de desempeo, en este caso el R2.

Para implementar esto, primero divid mis datos en un conjunto de entrenamiento y otro de prueba, con el fin de entrenar el modelo y luego evaluar qu tan bien generaliza a datos nuevos. Despus defin mi variable de salida como el PRECIO_MEDIO y elimin columnas que no eran directamente predictoras como ID_ENTIDAD y ANIO.

Primero apliqu el mtodo Forward Selection, que comienza con pocas variables y va agregando una a una, siempre escogiendo la que ms mejora el modelo. Esto me permiti obtener un primer subconjunto de predictores relevantes.

Con este mtodo, el modelo se qued con las siguientes caractersticas: PRODUCCION_CARNE, VALOR_PRODUCCION, Ganado Bovino, INTERACCION_CABEZAS_PRODUCCION, Nuevo Len, Tabasco,Tlaxcala e INTERACCION_CARNE_VALOR.

Luego, sobre ese mismo subconjunto, utilic el mtodo Backward Selection, que hace lo contrario: parte de varias variables y va eliminando las menos tiles, hasta quedarse con un conjunto ms reducido y bueno, al final se qued con estas variables: PRODUCCION_CARNE, VALOR_PRODUCCION, Ganado Bovino, INTERACCION_CABEZAS_PRODUCCION, e INTERACCION_CARNE_VALOR.

Por ltimo, compar el desempeo de ambos modelos con la mtrica R2 para validar si realmente la reduccin de variables aport mejoras.

En el caso del modelo Forward, obtuve un R2 de aproximadamente 0.2495. Por otro lado, en el modelo Backward, el R2 fue ligeramente menor (0.2345). Esto significa que al reducir el nmero de variables a cinco, el modelo explica cerca del 23.4% de la variabilidad.

Todo esto me muestra que las variables que seleccion tienen cierta capacidad para explicar el Precio Medio, pero tambin que existen otros factores no considerados en el modelo que influyen en gran medida en esta variable.
